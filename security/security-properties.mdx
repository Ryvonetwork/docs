---
title: "Security Properties"
description: "The security properties of Agon V4 that matter most."
---

Agon V4 tightened the replay and account-binding edges that matter most for long-lived lanes and offchain settlement.

## Signed Domain Separation

`message_domain` is derived inside the program from a fixed Agon tag, the `program_id`, and the `chain_id`. The deployment-scoped signing domain is therefore enforced by the protocol itself rather than trusted to bootstrap tooling.

## Distinct Chain IDs

Mainnet, devnet, testnet, and localnet all use distinct chain IDs, so the derived message domain changes across environments.

## Permanent Lanes

By removing close and reopen lane cycles, V4 removes a whole class of replay complexity. Old commitments cannot be replayed into a reopened lane instance because reopened lane instances no longer exist as a concept.

## Canonical PDA Checks

Settlement paths validate canonical participant and `channel-v2` PDAs, not just arbitrary deserializable program-owned accounts.

## Monotonic Cumulative Settlement

Every unilateral and cooperative target must advance cumulative state monotonically. Stale replay attempts fail because the protocol only accepts forward movement.
