---
title: "StandardX Mode"
description: "Accept payments from standard x402 buyers and pay standard x402 merchants — no Agon account required on the other side."
---

**StandardX** is Agon's term for full x402 interoperability — paying or accepting payment with any buyer or merchant that implements the standard x402 protocol, not just Agon.

<Info>
  **StandardX is always enabled.** Every Agon merchant automatically accepts standardx buyers — no extra configuration. Agon acts as the x402 facilitator, co-signing as `feePayer` and broadcasting on-chain. A **$0.001 facilitation fee** is deducted from your Agon balance per transaction.
</Info>

## The Four Payment Cases

| # | Buyer | Merchant | Mechanism |
|---|-------|----------|-----------|
| 1 | Agon-native | Agon-native | Off-chain settlement (fastest, default) |
| **2** | **Agon-native** | **Standard x402** | Agon acts as buyer proxy, sends `PAYMENT-SIGNATURE` |
| **3** | **Standard x402** | **Agon-native** | Agon acts as facilitator, co-signs & broadcasts |
| 4 | Non-Agon | Non-Agon | Rejected (no merchant balance to cover $0.001 fee) |

---

## Case 3: Accept StandardX Buyers (Agon merchant)

Your Agon-powered API automatically accepts any standard x402 buyer.

### Flow

1. StandardX buyer requests your endpoint (no `X-AGON-TOKEN`)
2. Your SDK responds with `402 + PAYMENT-REQUIRED` (standard x402 `PaymentRequirements`)
3. Buyer creates a partial Solana `transferChecked` tx — signed but **not broadcast**
4. Buyer retries with the `PAYMENT-SIGNATURE` header (base64 partial tx)
5. Your SDK calls Agon's `POST /platform/sponsor-tx`
6. Agon validates the tx, co-signs as `feePayer`, broadcasts, and confirms
7. Your endpoint receives `200 + PAYMENT-RESPONSE` with the tx signature

<Warning>
  **StandardX buyers must hold on-chain USDC.** They are NOT using Agon's custodial balance. They need a funded Solana wallet with USDC before calling your API.
</Warning>

### Merchant setup (no extra config needed)

```typescript
import { AgonPlatformCore } from "@agonx402/platform";

const core = new AgonPlatformCore({
  agonUrl: "https://api-devnet.agonx402.com",
  platformKey: process.env.AGON_PLATFORM_KEY!,
  pricing: 1000, // $0.001 USDC
  // Optional: route standardx payments to a specific wallet
  // walletAddress: "YourSolanaWalletAddressBase58",
});
```

### Middleware example (Express)

```typescript
import express from "express";
import { AGON_HEADERS } from "@agonx402/types";

const app = express();

app.use("/api", async (req, res, next) => {
  const price = await core.calculatePrice(req);
  const resource = `${req.protocol}://${req.get("host")}${req.originalUrl}`;

  // StandardX buyer: PAYMENT-SIGNATURE header
  const paymentSignature = core.extractPaymentSignature(req.headers);
  if (paymentSignature) {
    const paymentRequiredHeader = await core.buildStandardxPaymentRequiredHeader(price, resource);
    const result = await core.sponsorStandardxTx(paymentSignature, paymentRequiredHeader, price);
    res.setHeader(AGON_HEADERS.PAYMENT_RESPONSE, Buffer.from(JSON.stringify(result)).toString("base64"));
    return next();
  }

  // Agon-native buyer: X-AGON-TOKEN header
  const consumerToken = core.extractConsumerToken(req.headers);
  if (!consumerToken) {
    const { status, body } = core.buildPaymentRequiredResponse(price);
    const standardxHeader = await core.buildStandardxPaymentRequiredHeader(price, resource);
    res.setHeader(AGON_HEADERS.PAYMENT_REQUIRED, standardxHeader);
    return res.status(status).json(body);
  }

  // ...continue Agon-native authorize/consume flow
});
```

## Case 2: Pay StandardX Merchants (Agon buyer)

When an Agon consumer calls a non-Agon merchant, Agon's proxy acts as the standard x402 client — no consumer-side code changes needed.

### Flow

1. `agon.fetch()` hits the non-Agon merchant → gets `402 + PAYMENT-REQUIRED`
2. Consumer SDK calls `POST /proxy` with the merchant URL and `PAYMENT-REQUIRED` value
3. Agon checks spending controls, reserves the consumer's balance
4. Agon constructs a partial `transferChecked` tx from its custodial wallet, partially signs it — **does not broadcast**
5. Agon sends the standard `PAYMENT-SIGNATURE` header to the merchant
6. The merchant's facilitator broadcasts the tx and pays SOL gas
7. Agon settles the consumer's ledger on a `200` response

```typescript
import { AgonClient } from "@agonx402/client";

const agon = new AgonClient({ baseUrl: "https://api-devnet.agonx402.com", apiKey: "..." });

// Works transparently with any x402 merchant
const res = await agon.fetch("https://any-x402-merchant.com/api/data");
```
