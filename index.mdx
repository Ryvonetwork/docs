Agon Protocol is a **permissionless cooperative clearing protocol** for machine payments on Solana.

The thesis is simple: agentic payments do not scale if every interaction has to pay a high constant settlement cost. Agon replaces that fee-per-payment model with **cooperation, deferred settlement, and cumulative commitments**.

Instead of forcing every API call or agent action to settle exactly onchain, Agon lets participants:

- register once as permanent participants
- open permanent directed lanes
- advance cumulative offchain obligations
- settle the latest state directly, in bundles, or cooperatively in clearing rounds

<Tip>
Agon's main claim is not just “multilateral clearing.” The core claim is that **trust, reputation, collateral, and cooperation** can scale machine payments more effectively than exact per-payment fee models.
</Tip>

## What V4 Changed

- Permanent participants: one wallet maps to one participant identity for the lifetime of the deployment.
- Permanent lanes: one directed lane exists per `payer_id + payee_id + token_id`.
- V4 messages: `agon-cmt-v4` and `agon-round-v4` remove close/reopen complexity and keep the signed format compact.
- In-program `message_domain`: replay separation is derived inside the protocol instead of trusted to bootstrap tooling.
