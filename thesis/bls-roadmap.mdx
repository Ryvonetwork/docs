---
title: "BLS Roadmap"
description: "How aggregate signatures would expand cooperative-clearing capacity."
---

BLS is a roadmap item for the cooperative-clearing path. V4 does not depend on it.

Today the protocol already gets meaningful compression from latest-commitment settlement, payee-side bundle settlement, and cooperative rounds signed with Ed25519. Those flows are live, measurable, and sufficient to demonstrate the core model.

Where BLS matters is round density. In the current design, participant signatures and signer metadata consume a large share of the transaction budget for multilateral rounds. Aggregate signatures would reduce that overhead, which would allow more participants and more lane updates to fit before transaction bytes become the main constraint.

That makes BLS valuable because multilateral clearing becomes more compelling as the signed graph gets larger. It does not change the basic protocol model. It increases the capacity of the densest settlement mode. The generated synthetic benchmarks in this site show both the current Ed25519 limits and the projected headroom under aggregate signatures.
