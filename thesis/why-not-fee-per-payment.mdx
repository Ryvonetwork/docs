---
title: "Why Not Fee Per Payment"
description: "Why exact payment on every interaction hits a constant-cost floor."
---

Exact payment per interaction is simple to reason about because each request is settled independently. The downside is that every small payment inherits the same settlement overhead: a signing path, a transaction path, a liveness requirement, and a fee floor.

That model can be acceptable for consumer transfers or infrequent API calls. It becomes much less attractive when an agent makes thousands or millions of low-value requests. At that point, the cost of settlement starts to dominate the value of the underlying work.

Agon takes a different approach. The hot path is a signed cumulative update, not a settlement transaction. Settlement happens later, and only the newest state matters. This is why Agon benchmarks itself against an exact-payment x402-style baseline. The comparison is not ideological. It is a way to show how many onchain settlements are avoided when a repeated relationship is treated as a clearing problem instead of a sequence of isolated payments.

This tradeoff is most useful when the same parties interact repeatedly and still want public, non-custodial settlement without paying full settlement overhead on every call.
