---
title: "Why Not Fee Per Payment"
description: "Why exact payment on every interaction hits a constant-cost floor."
---

Exact payment per interaction is simple to explain because each request settles independently.

The problem is that every request inherits the same fixed settlement cost, even when the underlying work is tiny.

## Where The Cost Comes From

Per-payment settlement usually means that each interaction needs its own settlement transaction and its own fee. That is fine when payments are large or infrequent. It becomes inefficient when an agent is making many very small requests to the same provider.

## How Agon Changes The Cost Model

Agon uses deferred settlement. Payment execution happens first through signed cumulative updates, and settlement happens later onchain.

This does not eliminate cost. It changes where the cost appears. Instead of paying the full settlement cost every time, the relationship pays for occasional settlement.

## Why The Comparison To x402 Still Matters

Agon often compares itself to an exact-payment x402-style baseline because that is the cleanest way to show the difference in transaction count.

The point of that comparison is not that x402 is "bad." It is that exact settlement per request creates a constant-cost floor. Agon is designed for the cases where that floor is too high relative to the value of each interaction.
