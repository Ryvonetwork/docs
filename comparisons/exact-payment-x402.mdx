---
title: "Exact-Payment x402"
description: "Why Agon compares itself against exact per-payment settlement."
---

Agon does not reject x402. It questions exact settlement on every interaction as the dominant primitive for high-frequency machine commerce.

If every API call settles exactly, every call inherits transaction overhead, a fee floor, and a liveness bottleneck.

Agon therefore treats exact per-payment x402 as a baseline to measure against. The benchmark question is simple: how many exact-payment transactions would this scenario require if every call had to settle individually? That is the baseline shown in Agon's benchmark artifacts.
