## How to read this

- **Underlying Payments** is the number of individual payments the scenario represents if every payment were settled on-chain independently.
- **Settlement Txs** is the number of actual on-chain transactions Ryvo used to settle all of them.
- **Compression** is the ratio of the two, and the second number is total transactions saved.
- **Signed Message Bytes** is the Ed25519-signed body size, not the full transaction.

The clearing-round scenarios (3, 4, 5) intentionally set the cumulative targets to large values to illustrate that Ryvo's compression does not care about the size of the underlying obligations. The settlement cost is a function of the signed round, not of the payments it represents.
